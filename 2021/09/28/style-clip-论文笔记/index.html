StyleCLIP:Text-Driven Manipulation of StyleGAN Imagery 论文笔记

Posted on 2021-09-28

Introduction

将最近比较火的StyleGAN和Contrastive Language-Image Pre-training Models(CLIP)进行结合，由于CLIP用了非常多的image和text数据进行训练，用CLIP作为约束能够使StyleGAN生成更丰富的、text-guided的图像。CLIP是用了对比学习的思路，把text和image encode到一个joint representation，加入text的原因是natural language is able to express a much wider set of visual concepts。

其实CLIP就相当于一个见过很多种图像的辅助分类器，能guide stylegan生成符合text prompt的图像。就像之前的文章， 用一个辅助分类器，来让generator生成正确domain的图像，两者有异曲同工之妙，只是如今的CLIP更加的强大。

加入了CLIP的StyleGAN，只需要输入text prompt就能完成manipulation，并且还能完成unmapped的direction，无需更多人力和标注数据（zero-shot）

Motivation

首先，CLIP能够estimate the semantic similarity between a given text and an image，这就让它能够充当一个auxiliary classifier加入到GAN中，来约束generator生成正确的图像，所以本质的思想和以往的manipulation framework无大区别。

StyleGAN+CLIP就是利用StyleGAN能生成高质量图像的性质和CLIP经过庞大数据训练、拥有丰富的multi-domain semantics的性质。

其次，这里有一个新的概念，叫text-guided image generation and manipulation，其实早先的StarGAN也符合这个定义，输入图像和target domain，而target domain就是和一系列text prompt绑定在一起的。而现在比较多的方法是train an encoder to map text into the StyleGAN latent space，这样就可以根据text来生成图像。这篇文章提到一点，该文章只针对manipulation不针对generation，因为generation更加复杂和困难。

最后，文章提出了三个方法实现基于StyleGAN+CLIP的text-guided image manipulation：

  1. Text-guided latent optimization
  2. A latent residual mapper（I2I translator）
  3. A method for mapping a text prompt into an input-agnostic(global) direction in StyleGAN’s style space

Proposed Methods

Latent Optimization

核心思想：A given latent code of an image in StyleGAN’s W+ space is optimzed by minimizing a loss computed in CLIP space.

补充一些知识，StyleGAN用一个mapping network将初始的Z latent space映射到一个W latent space，在不使用style-mixing的情况下，stylegan generator每一层加入的w都是一样的。W+空间是指，这些generator每一层加入的w可以是不一样的，因为不同层加入的w控制的属性被stylegan大致分为了fine、medium、coarse三类，也就是说，不同层加入w控制了不同的属性。

Latent Mapper

核心思想：A Mapping network is trained， for a specific text prompt t， to infer a manipulation step M(w) in the W+ space. 从优化code到优化一个mapper。

通过CLIP，作者发现，text prompts are not limited to a single attribute at a time，意思是一个prompt能同时修改几个属性，其实这也证实了用mapper会有很严重的scalability的问题，因为属性的组合可以非常多，每一个组合对应一个promt，那么会需要训练非常多的mapper。

并且，因为一个prompt能对应多个属性的组合，所以容易出现entanglement的现象，it sometimes falls short when a fine-grained disentangled manipulation is desired. 其实这里也暗示出了CLIP的一个局限，text prompt在fine-grained的attribute上面存在entanglement。因为一般disentanglement属性都是由这个auxiliary classifier提供的。

最后，使用latent mapper是local的，因为*The direction of the manipulation step may vary depending on the starting position in W+*，而作者想要找到一个global的direction，有点像InterFace中的向量n，每一张图都能用该向量来实现manipulation。

Global Direction

核心思想：Mapping a text prompt into a single, global direction in StyleGAN’s style space S.

论文中说，这个style space S拥有更加disentangled的性质，所以就只专注于找一个global direction了。

具体方法：

  1. Use the CLIP text encoder to obtain a vector △t in CLIP’s joint language-image embedding
  2. Map this vector into a manipulation direction △s in Style Space S

下面进行细节介绍

From natural language to △t

核心思想：Prompt engineering is then applied to produce the average embeddings for the target and the neutral class, and the normalized difference between the two embeddings is used as the target direction △t

这里也暴露出论文一个比较大的问题，说着叫global direction，但还是需要一个neutral class作为参考来计算△t，那如果进行manipulate的图像不是neutral class，使用这个方法好不好使呢？所以这个和InterFaceGAN中的思路比较，就更基础一点，前者用了SVM寻找超平面，后者直接用average来当作neutral class，那这个就是最原始的vector arithmetic property。

这里的prompt engineer就是一个regularization的作用，一些text prompt拥有相同的语义，就把它们归为一类，然后计算一个平均值，就代表它们的embedding。

得到△t了之后，论文有一个重要假设：we expect directions on the T and I manifolds that correspond to the same semantic changes to be roughly collinear (i.e., have large cosine similarity) and nearly identical after normalization

意思就是normalization之后，△t就是△i了。

Discussion

这篇文章问题挺多的：

  1. Mapper存在scalability的问题，一个text prompt就要一个mapper；
  2. CLIP本身可能在一些fine-grained的属性上存在entanglement；
  3. Global Direction的寻找方法非常朴素，并且用最原始的average来寻找一个全局的start point。

除此之外，还有些作者提出来的局限:

  1. it cannot be expected to manipulate images to a point where they lie outside the domain of the pretrained generator
  2. text prompts在CLIP的embedding中如果没有对应的很多的图像，也无法很好的产生一个视觉上的manipulation
  3. 一些剧烈的变化也很难达到

第三点有点像一个out-of-domain的问题，后面一篇StyleGAN-NADA提了一些思路。
